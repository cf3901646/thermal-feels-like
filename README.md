# 🌡️ 体感温度计 (Thermal Comfort Meter) v2.0

一款专为关注“人类真实感受”而设计的**双重（室内/室外）体感温度分析与预报系统**。不只是简单地展示天气预报，而是通过物理和热力学模型，计算出你在室内和户外烈日下最真实的体感温度。

本软件已完成 Capacitor 移动端原生沉浸式适配，并编译生成了 Android 安装包。

---

## ✨ 核心亮点与特色功能

1. **🏠 室内外双重体感对比**
   - 传统天气软件只提供单一的“室外体感”或“气温”。
   - 本软件在同一屏幕上直观对比**室内体感温度**与**室外体感温度**，直接告诉你当前开窗通风是否会更凉快，或是应该关窗保温。
2. **📈 24小时精细体感变化曲线**
   - 包含过去 3 小时到未来 20 小时（共 24 小时）的超长跨度体感曲线。
   - 室内/室外双曲线采用高阶贝塞尔曲线平滑绘制，视觉丝滑。
3. **📅 7天温度区间走势图**
   - 基于 Chart.js 构建，以渐变填充区间的形式展示未来 7 天室内与室外的体感冷暖走势。
4. **🗺️ 高清高德地图 + 极速本地城市库**
   - 内嵌高清中文高德地图，支持直接在地图上选点定位获取天气。
   - 自建高精本地中国城市数据库，支持汉字、拼音、首字母简拼（如 `bj` 检索北京）极速模糊检索。
   - 剥离“市/区/县”等行政后缀，并结合 Open-Meteo 全球联网 Geocoding 进行兜底检索，实现 8 公里以内的智能去重。
5. **📱 移动端原生沉浸式适配**
   - 支持 Capacitor 原生插件，完美冲入 Android 状态栏并将其设为全透明。
   - 针对 WindowInsetsCompat 进行了精细的非遮挡排版设计。

## 🆚 与传统天气软件有什么不同？

大部分天气软件只是将气象站的数据直接搬运到屏幕上。然而，气象站测得的“气温”与人体“真实感受”存在巨大差异。

| 比较维度 | 传统天气软件 | 🌡️ 体感温度计 v2.0 |
| :--- | :--- | :--- |
| **空间维度** | 只关注户外（室外），忽略室内。 | **双重关注**。针对你 90% 时间所处的室内环境进行热动力模拟。 |
| **太阳暴晒修正** | 忽略暴晒的影响，或者体感公式对阳光不敏感。 | **精准太阳辐射增益**。将实时的短波太阳辐射（Shortwave Radiation）融入室外体感计算，烈日下体感骤升，完美模拟阳光直射的灼热感。 |
| **室内空气流动** | 无法评估。 | **动态环境补偿**。夏季通风时模拟风冷留存；冬季关闭窗户时模拟防风保温，并考虑电器与人体产生的微量热量。 |
| **建筑物热效应** | 认为室内温度与室外完全实时同步。 | **墙体热惰性（Thermal Mass）**。基于过去 12 小时的滑动平均温度模拟墙体的热阻隔与蓄热效应，温度过渡更平滑、真实。 |

## 🧠 体感算法的核心考量因素

本软件的体感温度计算基于经典的 **Steadman 体感温度模型（Steadman's Apparent Temperature）**，并在此基础上进行了多项深度优化：

### 1. 基础物理因子
* **空气温度** ( $T_a$ )：热力学的基本参数。
* **空气湿度** ( $RH$ )：决定人体的蒸发散热效率。湿度大时，高温会显得更闷热（汗液无法蒸发），低温会显得更湿冷（水汽导热性强）。
* **风速** ( $ws$ )：加速皮肤表面的对流散热，产生风冷效应。

---

### 2. 室内体感温度算法 (calcIndoorAT)

我们构建了**灵动权重与环境补偿模型**：

**🏢 墙体热惯性 (Thermal Mass)**
室内气温由室外实时空气（占 70%）和墙体蓄热（占 30%，取过去 12 小时的滑动平均温度）共同决定，平滑了瞬时的气温波动。计算公式如下：

$$
T_{\text{indoor}} = 0.7 \times T_a + 0.3 \times T_{\text{wall}}
$$

**🍃 夏季通风模式** (当气温 $T_a \ge 25\text{°C}$ 时)
假定室内采取通风措施，能够保留 40% 的室外风力。算法将补偿流失的 60% 风冷收益，上限为 $3.0\text{°C}$：

$$
EnvBonus = \min(ws \times 0.6 \times 0.5, 3.0)
$$

**🔥 冬季保温模式** (当气温 $T_a \le 15\text{°C}$ 时)
假定窗户紧闭。室内完全屏蔽了寒风，因此“退回”风冷效应造成的降温，并额外加上 $+1.0\text{°C}$ 以模拟人员活动、电器产生的微量热量。

---

### 3. 室外太阳暴晒体感修正 (calcOutdoorATWithSun)

传统的表观温度（Apparent Temperature）公式没有考虑太阳直射。本算法根据白天光照强度和气温，进行了太阳辐射补偿：

**☀️ 太阳辐射强度** ( $Rad_{\text{MJ}}$ )：以兆焦耳/平方米/小时 ( $MJ/m^2/h$ ) 为单位。

**🌡️ 温度调节因子** ( $Factor_{\text{temp}}$ )：低气温下暴晒增益减弱，公式为：

$$
Factor_{\text{temp}} = \max\left(0.4, \min\left(\frac{T_a - 10}{30}, 1.0\right)\right)
$$

**⛱️ 曝晒体感增益**：

$$
\Delta T_{\text{sun}} = \min(Rad_{\text{MJ}} \times 1.6 \times Factor_{\text{temp}}, 8.0)
$$

> [!NOTE]
> 在 $1000\text{W/m}^2$ (约 $3.6\text{ MJ/h}$ ) 的强日光下，可为户外人体带来高达约 $5.7\text{°C}$ 的体感温度上升，上限为 $8.0\text{°C}$。

## 🛠️ 技术栈与项目结构

- **前端核心**：纯原生 HTML5 / CSS3 / JavaScript (ES6+)，无繁重的第三方框架依赖，极速加载。
- **数据可视化**：[Chart.js](https://www.chartjs.org/) 提供动态且美观的渐变折线图。
- **地图服务**：[Leaflet.js](https://leafletjs.com/) 深度整合高清中文高德地图。
- **移动端封装**：[Capacitor](https://capacitorjs.com/)，实现一次开发，跨平台（Web, Android, iOS）部署。

### 📂 主要文件说明
- `index.html`：主页面的 HTML5 结构设计。
- `style.css`：极具美感、符合现代潮流的毛玻璃（Glassmorphism）暗黑/柔和系 CSS 样式。
- `main.js`：核心业务逻辑与体感算法计算。
- `china_cities.js`：包含数千个中国县级及以上城市的本地经纬度数据库。
- `体感温度计_2.0.APK`：已编译的 Android 原生安装包，可直接在真机上安装运行。

## 🚀 运行与开发

### Web 运行
你可以直接双击 `index.html` 在浏览器中运行，或者启动一个本地 Web 服务：
```bash
# 使用 Python 快速启动
python -m http.server 8000
```
然后访问 `http://localhost:8000`。

### 移动端构建与安装
1. 项目中已包含编译好的 `体感温度计_2.0.APK`，你可以直接将其拷贝到 Android 手机上安装。
2. 若要自己重新编译：
   ```bash
   npm install
   npx cap sync
   npx cap open android
   ```
   并在 Android Studio 中点击 Run 编译你自己的版本。
